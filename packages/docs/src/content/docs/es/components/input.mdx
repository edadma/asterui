## API

### `Input`

| Propiedad | Descripción | Tipo | Predeterminado |
|----------|-------------|------|---------|
| `type` | Tipo de input HTML | `'text' \| 'password' \| 'email' \| 'number' \| 'date' \| 'datetime-local' \| 'week' \| 'month' \| 'tel' \| 'url' \| 'search' \| 'time'` | `'text'` |
| `size` | Tamaño de input | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | - |
| `color` | Variante de color de input | `'neutral' \| 'primary' \| 'secondary' \| 'accent' \| 'info' \| 'success' \| 'warning' \| 'error'` | - |
| `status` | Estado de validación (establece aria-invalid) | `'error' \| 'warning'` | - |
| `ghost` | Variante ghost (transparente) | `boolean` | `false` |
| `bordered` | Mostrar borde (establece en false para eliminar) | `boolean` | `true` |
| `allowClear` | Mostrar botón de limpiar cuando el input tiene valor | `boolean \| { clearIcon?: ReactNode }` | `false` |
| `onClear` | Callback cuando se hace clic en el botón de limpiar | `() => void` | - |
| `prefix` | Icono o elemento de prefijo (dentro del input) | `React.ReactNode` | - |
| `suffix` | Icono o elemento de sufijo (dentro del input) | `React.ReactNode` | - |
| `addonBefore` | Texto/elemento antes del input (fuera, usando label de DaisyUI) | `React.ReactNode` | - |
| `addonAfter` | Texto/elemento después del input (fuera, usando label de DaisyUI) | `React.ReactNode` | - |
| `floatingLabel` | Texto de etiqueta flotante (usa floating-label de DaisyUI) | `string` | - |
| `mask` | Patrón de máscara de input. Usa # para dígitos, A para letras, * para alfanumérico | `string` | - |
| `maskPlaceholder` | Carácter de marcador de posición mostrado en la máscara | `string` | `'_'` |
| `errorId` | ID del elemento de mensaje de error (para aria-describedby) | `string` | - |
| `disabled` | Estado deshabilitado | `boolean` | `false` |
| `placeholder` | Texto de marcador de posición | `string` | - |
| `value` | Valor del input | `string` | - |
| `onChange` | Manejador de evento de cambio | `(e: React.ChangeEvent<HTMLInputElement>) => void` | - |
| `className` | Clases CSS adicionales | `string` | - |

## Accesibilidad

- Usa elemento `<input>` nativo para soporte de teclado adecuado
- El estado deshabilitado se comunica apropiadamente a tecnologías asistivas
- El botón de limpiar tiene `aria-label` para lectores de pantalla
- Usa `status="error"` para establecer `aria-invalid` automáticamente
- Usa `errorId` para vincular el input al mensaje de error a través de `aria-describedby`
- Los estados de enfoque son claramente visibles para navegación por teclado
