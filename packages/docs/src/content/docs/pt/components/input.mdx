## API

### `Input`

| Propriedade | Descrição | Tipo | Padrão |
|----------|-------------|------|---------|
| `type` | Tipo de input HTML | `'text' \| 'password' \| 'email' \| 'number' \| 'date' \| 'datetime-local' \| 'week' \| 'month' \| 'tel' \| 'url' \| 'search' \| 'time'` | `'text'` |
| `size` | Tamanho do input | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | - |
| `color` | Variante de cor do input | `'neutral' \| 'primary' \| 'secondary' \| 'accent' \| 'info' \| 'success' \| 'warning' \| 'error'` | - |
| `status` | Status de validação (define aria-invalid) | `'error' \| 'warning'` | - |
| `ghost` | Variante ghost (transparente) | `boolean` | `false` |
| `bordered` | Mostrar borda (defina como false para remover) | `boolean` | `true` |
| `allowClear` | Mostrar botão de limpar quando input tem valor | `boolean \| { clearIcon?: ReactNode }` | `false` |
| `onClear` | Callback quando botão de limpar é clicado | `() => void` | - |
| `prefix` | Ícone ou elemento de prefixo (dentro do input) | `React.ReactNode` | - |
| `suffix` | Ícone ou elemento de sufixo (dentro do input) | `React.ReactNode` | - |
| `addonBefore` | Texto/elemento antes do input (fora, usando label DaisyUI) | `React.ReactNode` | - |
| `addonAfter` | Texto/elemento depois do input (fora, usando label DaisyUI) | `React.ReactNode` | - |
| `floatingLabel` | Texto do rótulo flutuante (usa floating-label DaisyUI) | `string` | - |
| `mask` | Padrão de máscara de input. Use # para dígitos, A para letras, * para alfanumérico | `string` | - |
| `maskPlaceholder` | Caractere placeholder mostrado na máscara | `string` | `'_'` |
| `errorId` | ID do elemento de mensagem de erro (para aria-describedby) | `string` | - |
| `disabled` | Estado desabilitado | `boolean` | `false` |
| `placeholder` | Texto placeholder | `string` | - |
| `value` | Valor do input | `string` | - |
| `onChange` | Manipulador de evento de mudança | `(e: React.ChangeEvent<HTMLInputElement>) => void` | - |
| `className` | Classes CSS adicionais | `string` | - |

## Acessibilidade

- Usa elemento `<input>` nativo para suporte adequado de teclado
- Estado desabilitado é comunicado adequadamente para tecnologias assistivas
- Botão de limpar tem `aria-label` para leitores de tela
- Use `status="error"` para definir `aria-invalid` automaticamente
- Use `errorId` para vincular input à mensagem de erro via `aria-describedby`
- Estados de foco são claramente visíveis para navegação por teclado
